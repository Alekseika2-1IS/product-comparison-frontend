Курсовой проект по дисциплине «Разработка кода информационной системы».  
**Тема:** Интерфейс для сравнения товаров  
**Студент:** Крылов Алексей  
**Специализация:** Frontend-разработка

---

## Описание проекта

Веб-приложение для наглядного сравнения характеристик 2–3 товаров.  
Реализован полный CRUD (создание, чтение, обновление, удаление) для товаров.  
Данные хранятся в `json-server` (имитация backend-API).

---

## Функциональные возможности

- Просмотр каталога товаров в виде карточек.
- Выбор от 2 до 3 товаров для сравнения.
- Страница сравнения с таблицей характеристик.
- Добавление нового товара через форму.
- Редактирование и удаление товаров.
- Визуальное выделение различий в характеристиках.
- Сохранение выбранных товаров в `localStorage`.

## Установка и запуск

### 1. Установите Node.js
Скачайте и установите Node.js с официального сайта (рекомендуется версия LTS).  
После установки проверьте, что всё работает: откройте терминал (командную строку) и выполните:

node --version
npm --version
Должны отобразиться версии (например, v20.18.0 и 10.8.0).

2. Клонируйте репозиторий (или скопируйте папку проекта)
Если у вас ещё нет проекта, склонируйте его с GitHub:

git clone https://github.com/Alekseika2-1IS/product-comparison-frontend.git
cd product-comparison-frontend
Или просто перейдите в папку, где уже лежат все файлы.

3. Установите зависимости
Выполните команду:

npm install
Подождите, пока установятся все необходимые пакеты (React, Vite, json-server и др.).

4. Запустите сервер разработки (фронтенд)
В этом же терминале выполните:

npm run dev
После запуска вы увидите сообщение:

VITE v...  ready in ... ms
➜  Local:   http://localhost:5173/
Не закрывайте это окно – в нём работает ваше приложение.

5. Запустите json-server (имитация бэкенда)
Откройте новый терминал (отдельное окно командной строки), перейдите в ту же папку проекта и выполните:

npm run server
Должно появиться:

JSON Server started on PORT :3001
Endpoints: http://localhost:3001/products
Теперь мок-API доступен.

6. Откройте приложение в браузере
Перейдите по адресу: http://localhost:5173
Вы увидите каталог товаров.
Данные о товарах берутся из файла src/db.json.

Примечания:

Убедитесь, что порты 5173 и 3001 не заняты другими программами.

Если нужно остановить серверы, нажмите Ctrl+C в соответствующих терминалах.

Все изменения в коде автоматически обновляют страницу благодаря Vite.

Запуск тестов
В проекте реализованы модульные тесты для компонента ProductCard с использованием Jest и React Testing Library. Чтобы запустить тесты, выполните команду:

npm test
или напрямую:

npx jest
После выполнения вы увидите результат:

PASS  src/components/ProductCard/ProductCard.test.jsx
  ProductCard
    √ отображает название и цену товара (39 ms)
    √ вызывает onSelect при клике на чекбокс (26 ms)

Test Suites: 1 passed, 1 total
Tests:       2 passed, 2 total
Примечание: Для успешного прохождения тестов необходимо, чтобы все зависимости были установлены (см. шаг 3). Если вы вносите изменения в компонент, тесты помогут убедиться, что функциональность не нарушена.
